Declaring Regular Expressions

A screenshot of my validation code where I declare regular expressions, to screen the form input that the user has typed. These are assigned to variables so they can easily be passed into my validation function, which also makes the code cleaner, so should they need future modification its very straight forward. The first one checks for only text with spaces. The second one checks for only text but also allows new lines incase the user returns to a new line when they write their message. The third one is checking for only numbers with spaces allowed. Finally the last one is making a best effort attempt at checking for valid email addresses, although this is never going to be perfect.

My Validate Function

This is a screenshot of my validation function, it takes two arguments, id and index. First the function gets the value from the id which has been passed into it, trims away any whitespace assigns it to the variable val. Then a fail function is created to return the failure styling whenever it is called to prevent repeated code. Now the variable val is passed through each if statement in the validation function. If it fails at anypoint the appropriate error message is generated, which uses the index value as a key to identify which field has failed, which then gets returned and the fail function is called to apply the failed styling to the form. When the form values successfully pass through the entire validate function, any error message present is cleared and the success styling is applied, when every value of the form passes successfully the data gets submitted.

Call the Function

A screenshot of my validation code showing how I set an event listener to listen for the form submit. When the form submit button is clicked the function prevents the form from auto-submitting, it then calls on the validate function for each field of the form. It passes the form field id and an index value through the function and once each one successfully passes it allows the form to be submitted
