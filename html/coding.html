Animation Effects

A code-snippet of my CSS which I wrote to add a glass shine style hover effect to my portfolio project display. I created it as a Sass Mixin so I could easily reuse the effect.


@mixin fancy-shine  {
    &-img::before     {
        content: "";
        position: absolute;
        bottom: -20%;
        left: -90%;
        width: 150%;
        height: 4rem;
        z-index: 1;
        background-color: $white;
        opacity: 0.2;
        transform: rotate(45deg);
        transition: all 350ms ease-in;
    }
    &:hover &-img::before,
    &:focus &-img::before    {
        bottom: 100%;
        left: 40%;
    }
}

Defining Media Queries

A code-snippet of my Sass code which is used to define media queries, I can pass the screen size as an argument and then repeat the same code to multiple elements of my webpage. The Mixin will reference the Sass Map I have defined, check that the argument I supplied is valid and then construct the requested media query.


@mixin mqm($mq-breakpoint, $mq-breakpoints: $breakpoints-up)   {

    @if map-has-key($mq-breakpoints, $mq-breakpoint)           {
        $mq-breakpoint: map-get($mq-breakpoints, $mq-breakpoint);
    }

    @media screen and (min-width: #{$mq-breakpoint})           {
        @content;
    }
}

@mixin mqx($mq-breakpoint, $mq-breakpoints: $breakpoints-down) {

    @if map-has-key($mq-breakpoints, $mq-breakpoint)           {
        $mq-breakpoint: map-get($mq-breakpoints, $mq-breakpoint);
    }

    @media screen and (max-width: #{$mq-breakpoint})           {
        @content;
    }
}

Declaring Arrays

A code-snippet of my validation code which is used by my validate function to construct error messages. The first is a basic array containing string values for each of the form field names. The second is an object array containing property value pairs, these are for assigning the appropriate error to the field name, chosen from the first array.


const indexMsg   =
    [
    "First Name",
    "Last Name",
    "Email",
    "Phone Number",
    "Subject",
    "Message"
]

const errMsg    =
    {
    noContent   : " is required",
    hasInvalid  : " contains invalid characters",
    nameIsShort : " is too short, min 2 characters",
    telIsShort  : " is too short, min 11 digits",
    telIsLong   : " is too long, max 11 digits",
    subIsShort  : " is too short, min 4 characters",
    msgIsShort  : " is too short, min 20 characters",
    isLong      : " is too long, max 254 characters",
    notEmail    : " given is not a valid email",
    notTel      : " given is not a valid number"
}

Declaring Regular Expressions

A code-snippet of my validation code where I declare regular expressions, which are used to screen the form input that the user has typed. These are assigned to variables so they can easily be passed into my validation function, this also makes the code cleaner, so should they need future modification it is very straight forward. The first one checks for only text with spaces. The second one checks for only text but also allows new lines incase the user returns to a new line when they write their message. The third one is checking for only numbers, this is a very simple filter as the field is optional, which makes it easy for the user to supply a contact number if they wish. Finally the last one is making a best effort attempt at checking for valid email addresses, although this is never going to be perfect.


const
    regExpTx   = new RegExp(/^[a-zA-Z ]*$/),
    regExpMsg  = new RegExp(/^[a-zA-Z\s]*$/),
    regExpTel  = new RegExp(/^[0-9]*$/),
    regExpEm   = new RegExp(/^(([^<>()\[\]\\
    .,;:\s@"]+(\.[^<>()\[\]\\.,;:\s@"]+)*)|
    (".+"))@((\[[0-9]{1,3}\.[0-9]{1,3}\.[0-9]
    {1,3}\.[0-9]{1,3}])|(([a-zA-Z\-0-9]+\.)+
    [a-zA-Z]{2,}))$/);

My Validate Function

This is a code-snippet of my validate function, it takes the two arguments id and index. First the function gets the value from the id which has been passed into it, then trims away any whitespace from each end and assigns it to the variable val. Then a fail function is created to return the failed styling, whenever it is called, to prevent repeated code. Now the variable val is passed through each if statement in the validate function. If it fails validation at anypoint, the appropriate error message is generated and the fail function is called to apply the failed styles. This uses the index value as a key to identify which field has failed, which then gets returned along with the failed styling to the form. When the each of the form values successfully pass through the entire validate function, any error message present is cleared and the passed styling is applied. When each field value of the form passes successfully, the data is allowed to be submitted.


let validate  =  (id, index) => {
    let val   =  id.value.trim();
    let fail  =  ()     =>          {
        id.style.border = "2px solid red";
        failureIcon[index].style.opacity = "1";
        successIcon[index].style.opacity = "0";
    };
    if (val === "" && id !== phone)                 {
        errorMsg[index].innerHTML = indexMsg[index] + errMsg.noContent;
        fail();
        return;
    }
    if (!val.match(regExpTx) && id !== email && id !== message && id !== phone)	{
        errorMsg[index].innerHTML = indexMsg[index] + errMsg.hasInvalid;
        fail();
        return;
    }
    if (val.length < 2 && id === firstName)         {
        errorMsg[index].innerHTML = indexMsg[index] + errMsg.nameIsShort;
        fail();
        return;
    }
    if (val.length < 2 && id === surname)           {
        errorMsg[index].innerHTML = indexMsg[index] + errMsg.nameIsShort;
        fail();
        return;
    }
    if (!val.match(regExpEm) && id === email)       {
        errorMsg[index].innerHTML = indexMsg[index] + errMsg.notEmail;
        fail();
        return;
    }
    if (!val.match(regExpTel) && id === phone)      {
        errorMsg[index].innerHTML = indexMsg[index] + errMsg.notTel;
        fail();
        return;
    }
    if (val.length !== 0 && val.length < 11 && id === phone)  {
        errorMsg[index].innerHTML = indexMsg[index] + errMsg.telIsShort;
        fail();
        return;
    }
    if (val.length > 11 && id === phone)	        {
        errorMsg[index].innerHTML = indexMsg[index] + errMsg.telIsLong;
        fail();
        return;
    }
    if (val.length < 4 && id === subject)	        {
        errorMsg[index].innerHTML = indexMsg[index] + errMsg.subIsShort;
        fail();
        return;
    }
    if (!val.match(regExpMsg) && id === message)    {
        errorMsg[index].innerHTML = indexMsg[index] + errMsg.hasInvalid;
        fail();
        return;
    }
    if (val.length < 20 && id === message)          {
        errorMsg[index].innerHTML = indexMsg[index] + errMsg.msgIsShort;
        fail();
        return;
    }
    if (val.length > 254)    {
        errorMsg[index].innerHTML = indexMsg[index] + errMsg.isLong;
        fail();
        return;
    }
    errorMsg[index].innerHTML = "";
    id.style.border = "2px solid green";
    failureIcon[index].style.opacity = "0";
    successIcon[index].style.opacity = "1";
};

Call the Function

A code-snippet of my validation code showing how I set an event listener to wait for the form submit. When the form submit button is clicked the function prevents the form from auto-submitting, it then calls on the validate function for each field of the form. It passes the form field's id and an index value into the function, once each one successfully passes it allows the form to be submitted.


form.addEventListener('submit', (e) => {
    e.preventDefault();
    validate(firstName, 0);
    validate(surname, 1);
    validate(email, 2);
    validate(phone, 3);
    validate(subject, 4);
    validate(message, 5);
});
